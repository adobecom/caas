Introduction to CaaS - Content as a Service

The following provides a concise overview of the CaaS platform. For a comprehensive understanding, including detailed features and schedules, please refer to the CaaS Wiki page.

What is it?

CaaS, short for Content As A Service, is a platform designed to streamline content creation and management. Users can generate content in Milo or AEM, then transmit it to the CaaS backend service (Chimera) for storage in a database and retrieval.

The backend service's APIs facilitate the content retrieval, enabling display across various applications. All content is stored in JSON format, allowing for easy integration with different platforms.

How does it work?

From the users point of view there are two main components in the CaaS framework: the content creation and content consumption. Both, the creation and consumption of content can be done in either Milo or AEM platforms.

  • Content creation: This process involves generating content and transmitting it to the backend service. Each piece of content must be created individually in order to generate an unique content ID, then Sent to Caas so that it can then be rendered as either a card or a marquee.
  • Content consumption: Here, content is fetched from the backend service in JSON format and displayed within the application. Content can be presented in grid or carousel formats. The JSON response can include hundreds of content result items depending on the Card Collection configuration.

Why should I use it?

Some of the benefits of using CaaS are:

  • Enhanced efficiency in content creation and management
  • Facilitated content reuse across multiple applications
  • Real-time content updates
  • Consistent content display across diverse applications
  • Personalization options tailored to user preferences
  • Optimization for various devices
