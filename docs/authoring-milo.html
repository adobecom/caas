Introduction to CaaS - Authoring in Milo

The following is a brief introduction to the CaaS platform. For a more detailed information, features and schedules, please visit the CaaS Wiki page.

Create a new card

To create a new card in Milo, follow these steps:

  • Create a new page in Milo in the desired location
  • Open the Milo Library from the AEM sidekick on the top of the page
  • From the Blocks section, select Caas Card Metadata and copy and paste into your Milo document
  • Modify or create new metadata rows as needed

Available metadata options

The following are the most common metadata options for a basic card:

  • CardTitle
  • Description
  • CardImage
  • CTAText
  • CTAURL

For a more detail list of metadata options, please visit the CaaS Wiki page.

Send to CaaS

To send a card data to CaaS so that is available via the Chimera backend services, follow these steps:

  • Once the metadata is ready, click on the Preview button on the AEM sidekick bar.
    > A new blank page will be displayed in the main--milo-adobe.com preview server
  • In the preview page click on the Publish button
    > A new blank page will be displayed in the milo.adobe.com publish server
  • In the AEM sidekick, select Tools > Send To Caas.
    If everything is OK you will get to the confirmation page where you can:
    See All Properties - ProTip unique Ids can be seeing here withough the need to publish the card.
    Send to Caas - sends data to the caas backend (Chimera)

For a more detail list of metadata options, please visit the CaaS Wiki page.

Preview a Card in Different Styles

To preview a card in different styles using the Milo configurator you need to know the unique ID of the card you want to preview. The card's unique ID can be seeing during the Send To Caas workflow.

  • Open the Milo Caas Configurator tool
  • In the Basics tab select:
    Source: none
    Results Per Page: 1
    Total Cards to Show: 1
  • In the Cards tab, click on Featured Cards > Add
  • Paste the card's unique ID into the text field

For a more detail list of metadata options, please visit the CaaS Wiki page.
