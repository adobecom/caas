Introduction to CaaS

The following is a brief introduction to the CaaS platform. For a more detailed information, features an schedules, please visit the CaaS Wiki page.

What CaaS is a Content As A Service platform that allows users to create and manage content in a more efficient way. The content can be created in either Milo or AEM and then sent to the CaaS backend service (Chimera) where it gets stored in a database. The backend service APIs can then be used to retrieve the content and display it in any application.

How From the users point of view there are two main components in the CaaS framework: the content creation and content consumption. Both, the creation and consumption of content can be done in either Milo or AEM platforms.

  • The content creation process is responsible for creating the content and sending it to the backend service. Each invividual piece of content can be render as a card or a marquee.
  • The content consumption process is responsible for retrieving the content from the backend service as a JSON format and displaying it in the application. Content can be rendered as a grid or carousel format.

Why Some of the benefits of using CaaS are:

  • Content can be created and managed in a more efficient way
  • Content can be reused across multiple applications
  • Content can be updated in real-time
  • Content can be displayed in a consistent way across different applications
  • Content can be personalized based on user needs
  • Content can be optimized for different devices
